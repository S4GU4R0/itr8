# Fix Clipped Canvas Rendering Issue When Panning

## 1. Objective
Fix the viewport clipping issue where panning the canvas doesn't reveal additional content beyond what was originally loaded. The graph view should properly render and display all content when panned.

## 2. Impact Analysis

**Files to Modify:**
- Create: None
- Modify: `/Volumes/HD/Developer/Projects/itr8/src/components/graph-view.tsx`
- Delete: None

**Dependencies:** None

**Risk Assessment:** Low risk. The fix involves calculating proper bounds and applying them to the SVG rendering, which is a standard pattern for graph visualization libraries.

## 3. Implementation Details

### Phase 1: Calculate Graph Bounds
- **Goal**: Determine the total width and height needed to render all nodes and edges.
- **Key Changes**:
  - Add configurable padding constant (e.g., `const PADDING = 40`) for margins
  - Add logic to calculate the minimum X, maximum X, minimum Y, and maximum Y of all nodes and edges
  - Calculate total width = (max X - min X) + padding * 2
  - Calculate total height = (max Y - min Y) + padding * 2
- **File**: `graph-view.tsx` - Add new `useMemo` hook to compute bounds

### Phase 2: Set Proper SVG Dimensions
- **Goal**: Make the SVG size match the computed bounds instead of using 100% width/height.
- **Key Changes**:
  - Change SVG `width` and `height` attributes from `width="100%" height="100%"` to dynamic values: `width={bounds.width} height={bounds.height}`
  - Set `transformOrigin: "0 0"` (top-left of SVG coordinate system)
  - Apply transform in correct order: scale → translate
  - Transform formula: `transform: scale(${scale}) translate(${bounds.minX + padding}px, ${bounds.minY + padding}px)`
- **File**: `graph-view.tsx` - Update the SVG element rendering

### Phase 3: Update Offset State
- **Goal**: Ensure offset state works correctly with the new bounds system.
- **Key Changes**:
  - Update initial offset to `{ x: bounds.minX + padding, y: bounds.minY + padding }`
  - Adjust handleMouseDown to calculate drag start with new offset reference
  - Ensure handleMouseMove updates offset correctly
- **File**: `graph-view.tsx` - Update panning logic

## 4. Verification Strategy

**Testing Approach:**
1. Add test items with different connections to create a larger graph
2. Pan the view left/right/up/down to verify all content is revealed
3. Zoom in and out to ensure transform operations work correctly
4. Test edge cases: large graphs with many items, deep hierarchies

**Acceptance Criteria:**
- Panning left/right/up/down shows all nodes and edges in the graph
- Zoom in/out works smoothly without clipping
- Initial view centers the graph appropriately
- No visual artifacts or partial rendering
- Performance remains acceptable with graphs of various sizes
